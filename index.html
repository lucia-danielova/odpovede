2. Databázová úloha (Duplicity & Výkon)

Zadanie: SQL dotaz, ktorý vyberie všetky riadky z tabuľky "duplicates", ktoré majú v stĺpci "value" duplicitnú hodnotu.
-- Využitie Subquery s klauzulou HAVING pre presnú identifikáciu SELECT id, value FROM duplicates WHERE value IN ( SELECT value FROM duplicates GROUP BY value HAVING COUNT(*) > 1 ) ORDER BY value;
Strategický plán optimalizácie (Enterprise Scale): 1. Indexácia (B-Tree): CREATE INDEX idx_duplicates_value ON duplicates(value); - Zníženie časovej zložitosti skenovania tabuľky. 2. Query Rewrite (EXISTS klauzula): Pri obrovských objemoch je často efektívnejšie nahradiť 'IN' za korelovaný poddotaz s 'EXISTS', čo optimalizátoru databázy umožní lepšie plánovanie pamäte. 3. Asynchronous Batch Processing: Mazanie alebo úprava miliónov duplikátov naraz zamkne tabuľku (Table Lock). Riešením je spracovanie v dávkach (napr. LIMIT 1000) cez Cron.
